# 每日五领域信息候选

这个模块负责每天为五个主轴收集候选信息：

- 互联网
- AI
- 游戏
- 投资
- 写作

第一版不接付费 AI，不做自动结论，只做三件事：

1. 从公开搜索/RSS 拉取候选。
2. 按关键词、时效、来源、历史反馈打分。
3. 生成一个可勾选的私有清单，让你决定 `收录` 或 `移除`。

## 手动生成

```powershell
cd "D:\==我的学习库==\90_系统\daily_collect"
.\collect_daily_domains.ps1
```

指定日期：

```powershell
.\collect_daily_domains.ps1 --date 2026-06-22
```

输出位置：

```text
D:\==我的学习库==\00_入口收件箱\每日信息候选\YYYY-MM-DD.md
```

## 反馈学习

在当天的 `每日信息候选` 文件里，对每条候选勾选其中一个：

```text
- [x] 收录 `候选ID`
- [ ] 移除 `候选ID`
```

或者：

```text
- [ ] 收录 `候选ID`
- [x] 移除 `候选ID`
```

然后运行：

```powershell
cd "D:\==我的学习库==\90_系统\daily_collect"
.\apply_daily_feedback.ps1
```

反馈会写入私有文件：

```text
D:\==我的学习库==\90_系统\config\daily_collect_feedback.json
D:\==我的学习库==\00_入口收件箱\每日信息候选\筛选反馈.md
```

这些文件默认不提交到 GitHub。

## 筛选逻辑

采集脚本会按以下因素排序：

- 领域关键词匹配越多，分数越高。
- 命中标题党、广告、荐股等排除词，分数降低。
- 最近 1 到 3 天的信息优先。
- 你历史上经常收录的来源和关键词加分。
- 你历史上经常移除的来源和关键词降分。
- 已经出现过的链接会降权，减少重复。

## 来源配置

来源和关键词在这里维护：

```text
D:\==我的学习库==\90_系统\daily_collect\source_catalog.json
```

后续可以把某个领域的查询改得更窄，例如只看 `AI Agent 产品化`、`A股 游戏 公司 财报`、`写作 选题 表达`。
